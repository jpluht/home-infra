# Ansible Automation Overview

## Purpose

- Automate network and server configuration.
- Ensure consistency and repeatability.

## Components

- Playbooks: Define configuration tasks.
- Group_vars: Store device-specific variables.
- Templates: Generate configuration files (e.g., DHCP, VLANs).

## Best Practices

- Use Ansible Vault for secrets.
- Keep playbooks modular and reusable.
- Test changes in a safe environment before production.
